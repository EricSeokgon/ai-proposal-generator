# 제안서 자동 생성 에이전트 - 강의 스크립트 구성안

---

## 1. 제안서 자동 생성 에이전트를 만든 이유

### 마케팅/입찰 제안서 제작의 페인 포인트

- **매번 반복되는 제안서 구조 설계**
  - RFP를 받으면 평가기준 분석, 구조 설계, 장표 스크립트 작성을 매번 처음부터
  - 70~140장 분량의 제안서 구조를 짜는 데만 하루 이상 소요
  - Impact-8 Phase(HOOK→SUMMARY→INSIGHT→CONCEPT→ACTION→MANAGEMENT→WHY US→INVESTMENT) 설계가 반복적

- **디자인 퀄리티와 속도의 딜레마**
  - PPTX로 빠르게 만들면 디자인 퀄리티가 떨어짐
  - Figma로 고퀄리티를 만들면 시간이 3~5배 더 걸림
  - 디자이너에게 의뢰하면 커뮤니케이션 비용 + 대기 시간 발생

- **콘텐츠 기획력의 한계**
  - RFP를 분석해도 "차별화된 전략"을 도출하기 어려움
  - Win Theme(수주 전략 메시지)을 3개 만들어야 하는데 매번 고민
  - Action Title("타겟 분석" → "MZ세대 하루 SNS 55분, 릴스가 핵심")로 변환이 안 됨

- **리서치/데이터 수집의 비효율**
  - 시장 트렌드, 경쟁사 분석, KPI 벤치마크를 매번 수동으로 조사
  - "이 수치의 출처가 뭐야?"라는 질문에 답하기 어려움
  - KPI 목표를 설정하려면 업계 평균 데이터가 필요한데 찾기 번거로움

- **팀 협업/브레인스토밍의 어려움**
  - 혼자 기획하면 시야가 좁아짐
  - 팀원과 브레인스토밍하려면 회의 시간 조율 + 진행 비용
  - 전략가/크리에이터/데이터 분석가 관점을 동시에 얻기 어려움

### 이 문제를 에이전트로 해결하면?

- RFP/기획안 파일만 넣고 `/start` → **5단계 자동 실행** (분석→리서치→기획→제작→디자인)
- 3명의 AI 전문가(전략가·크리에이터·데이터분석가)가 **브레인스토밍 → 기획안 → 제안서** 풀 코스 지원
- **PPTX 납품용 + Figma 고퀄리티** 듀얼 출력 (PPTX 자동 생성 + HTML→Figma 변환)
- Impact-8 Phase 구조 + 30종 레이아웃 + 6종 테마 **자동 설계**
- Action Title, Win Theme, KPI 산출근거까지 **AI가 자동 기획**
- 시장 트렌드, 경쟁사 분석 **자동 리서치** (스킵 가능)
- QA 자동 검수 (겹침, 폰트, 컬러, 플레이스홀더 체크)

## 2. 제안서 자동 생성 에이전트 만든 과정

### Step 1: 문제 사항 정리
- 위에서 정리한 페인 포인트를 먼저 명확하게 정리
- "내가 원하는 건 이런 제안서다"라는 요구사항을 구체화
- **이 단계가 가장 중요** — 문제 정의가 명확할수록 에이전트의 결과물 품질이 올라감

### Step 2: RFP/기획안 데이터를 에이전트에 전달
- RFP(제안요청서) PDF 또는 기획안 DOCX 파일 준비
- `제안요청서/` 폴더에 파일 배치
- 선택: 디자인 레퍼런스(PPTX/이미지)를 `input/` 폴더에 추가

### Step 3: 프롬프트로 제안서 에이전트 제작 요청
```
RFP 분석부터 PPTX 제안서 자동 생성까지 해주는 에이전트를 만들어줘.
5단계 파이프라인으로 분석→리서치→기획→제작→디자인 변환을 자동화하고,
Impact-8 구조(HOOK~INVESTMENT)로 70~140장 제안서를 생성해야 해.
PPTX 납품용 + HTML→Figma 고퀄리티 디자인 듀얼 출력을 지원해야 해.
브레인스토밍 에이전트(3명 AI 전문가)와 연계도 지원해야 해.
```

### Step 4: 에이전트의 작업 흐름

에이전트는 프롬프트를 받으면 아래 순서로 작업을 진행합니다:

1. **파이프라인 설계** → 5단계 멀티 에이전트 파이프라인 구조 설계
2. **RFP 파서 구축** → PDF/DOCX 파일에서 텍스트, 평가기준, 요구사항 자동 추출
3. **분석 에이전트** → 평가기준 배점, Pain Point, 수주전략 자동 분석
4. **기획 에이전트 (4개 서브에이전트)** → 구조설계 + 스크립트 + 레이아웃 + 디자인 자동 생성
5. **제작 엔진 (slide_kit)** → 30종 레이아웃 + 6종 테마의 PPTX 렌더링 엔진 구축
6. **QA 검수 에이전트** → 겹침/폰트/컬러/플레이스홀더 자동 검수
7. **디자인 변환** → HTML→Puppeteer→JSON→Figma 플러그인 연동

### Step 5: 제안서 구조 설계의 핵심 — Impact-8 Framework

에이전트가 만드는 제안서는 **Impact-8 Phase 구조**로 설계됩니다:

```
Phase 0. HOOK (3-10p, 5%)     — 임팩트 있는 오프닝, 핵심 메시지
Phase 1. SUMMARY (3-5p, 5%)   — Executive Summary (의사결정자용 5분 요약)
Phase 2. INSIGHT (8-15p, 10%) — 시장 환경 + 문제 정의 + 숨겨진 니즈
Phase 3. CONCEPT (8-15p, 12%) — 핵심 컨셉 + 차별화 전략 + 경쟁 우위
Phase 4. ACTION (30-60p, 40%) — 상세 실행 계획 (핵심, 가장 비중 큼)
Phase 5. MANAGEMENT (6-12p, 10%) — 조직 + 운영 + 품질관리
Phase 6. WHY US (8-15p, 12%)  — 수행 역량 + 유사 실적
Phase 7. INVESTMENT (4-8p, 6%) — 투자 비용 + ROI
```

이 구조의 핵심:
- 제안서를 위에서부터 읽으면 **"왜 우리여야 하는지"가 자연스럽게** 전달됨
- Phase별 비중이 프로젝트 유형(마케팅/입찰/IT)에 따라 자동 조정
- Win Theme 3개가 Phase 전체에 일관되게 반영

### Step 6: slide_kit 렌더링 엔진

에이전트의 핵심 엔진은 **30종 레이아웃 + 6종 테마**를 지원하는 PPTX 렌더링 엔진입니다:

```
[왜 직접 엔진을 만들었는가?]
- python-pptx 라이브러리를 직접 쓰면 좌표 계산이 복잡하고 반복적
- 미리 만들어둔 함수를 조합하면 제안서 한 장을 ~20줄 코드로 생성 가능
- COLS(), FLOW(), PYRAMID(), TABLE(), KPIS() 등 도식화 함수가 핵심
- 테마를 apply_theme() 한 줄로 변경하면 전체 컬러가 바뀜

[30종 레이아웃 예시]
- FULL_BODY, TWO_COL, THREE_COL, FOUR_COL
- COMPARE_LR, KPI_GRID, STAT_ROW, TABLE_INSIGHT
- PROCESS_DESC, TIMELINE_DESC, PYRAMID_DESC
- KEY_VISUAL, ORG_CHART, GANTT, ICON_CARD_4 등

[6종 테마]
- warm_minimal (베이지), classic_blue (네이비), forest (그린)
- corporate (네이비), mono_black (블랙), soft_purple (퍼플)
```

### Step 7: 초안 기반으로 기능 추가 요청
- 초안이 나오면 직접 사용해보고 부족한 부분을 추가 요청
- 예시:
  - "Action Title이 너무 밋밋해, 숫자와 인사이트를 넣어줘"
  - "KPI에 산출근거를 자동으로 추가해줘"
  - "연속 3장 같은 레이아웃이면 자동으로 다른 걸로 변경해줘"
  - "QA에서 겹침이 발견되면 자동으로 재생성해줘"
  - "Figma로 바로 임포트할 수 있게 HTML→JSON 변환 플러그인 만들어줘"
  - "브레인스토밍 에이전트와 연결해서 기획부터 제안서까지 원스톱으로"
- 이렇게 반복하면서 원하는 기능을 지속적으로 추가하여 완성

## 3. /start 단축어 사용 방법

### /start란?

Claude Code에서 `/start`를 입력하면 제안서 제작 가이드가 자동으로 시작됩니다.
`.claude/commands/start.md` 파일에 정의된 워크플로우가 실행됩니다.

### /start 실행 화면

```
Claude Code 터미널에서 /start 입력
  ↓
안녕하세요! 👋 제안서 자동 생성 에이전트입니다.

🚀 진행 방식을 선택해 주세요:

  A. 브레인스토밍 → 제안서 (풀 코스)
     → 3명의 AI 전문가가 캠페인 기획 후 제안서 제작
     → 소요: 약 30~50분

  B. 제안서 바로 제작 (단독 실행)
     → RFP/기획안을 바로 분석하여 5단계 자동 실행
     → 소요: 약 15~30분

어떤 방식으로 진행할까요? (A 또는 B)
```

### /start가 동작하는 원리

```
.claude/commands/start.md   ← /start 입력 시 이 파일이 로드됨
  ├── 시작 인사 + 방식 선택 (A/B)
  ├── 제안서 유형 선택 (마케팅/입찰/기타)
  ├── 디자인 테마 선택 (6종)
  ├── STEP 1~5 자동 실행 가이드
  └── 각 STEP 완료 시 진행 상황 표시
```

### /start 이후 흐름 (B 방식 기준)

```

  ↓

  ↓

  [■□□□□] 20%
  RFP 분석 중...
  → 01_analysis.json 생성
  ↓
━━━━ STEP 2 🔍 리서치 ━━━━
  [■■□□□] 40%
  시장/경쟁 조사 중... (스킵 가능)
  → 02_research.json 생성
  ↓
━━━━ STEP 3 📐 기획 ━━━━
  [■■■□□] 60%
  4개 서브에이전트 순차 실행...
  → 03_plan.json 생성
  ↓
```

### /start가 안 될 때

| 원인 | 해결 |
|------|------|
| 작업 디렉토리 불일치 | `제안서 에이전트/` 폴더에서 Claude Code 실행 확인 |
| start.md 파일 없음 | `.claude/commands/start.md` 존재 확인 |
| Claude Code 미반영 | start.md 수정 후 Claude Code 재시작 |
| 대체 호출 | "제안서 만들어줘", "RFP 분석 시작" 등 자연어로도 동작 |

## 4. 제안서 자동 생성 에이전트 사용 방법

### 사용 방법 2가지

**방식 A — 브레인스토밍 → 제안서 (풀 코스)**
1. `/start` → A 선택
2. 3명의 AI 전문가(전략가·크리에이터·데이터분석가)가 캠페인 기획
3. 기획안 완성 후 자동으로 제안서 제작 연계 (STEP 1~2 스킵)
4. STEP 3(기획) → STEP 4(제작) → STEP 5(디자인 변환) 자동 실행
5. 소요: 약 30~50분

**방식 B — 제안서 바로 제작 (단독 실행)**
1. RFP/기획안을 `제안요청서/` 폴더에 넣기
2. `/start` → B 선택 → 5단계 자동 실행
3. 분석 → 리서치 → 기획 → 제작 → 디자인 변환
4. 소요: 약 15~30분

### 5단계 파이프라인

```
STEP 1 📄 분석    → RFP/기획안 분석 (타겟, 채널, 예산, 평가기준)
STEP 2 🔍 리서치   → 시장 트렌드, 경쟁사, KPI 벤치마크 조사 (스킵 가능)
STEP 3 📐 기획    → 구조 설계 → 장표 스크립트 → 레이아웃 → 디자인
STEP 4 🎨 제작    → PPTX 생성 (납품용) + 자동 검수
STEP 5 🎯 디자인  → Gemini HTML → Claude 워딩 수정 → Puppeteer → Figma 플러그인
```

### STEP 5 디자인 변환 상세 (6단계)

PPTX 납품 후 디자인 퀄리티를 높이고 싶을 때 진행하는 선택 단계입니다:

```
STEP 5-1: 디자인 레퍼런스 입력
  → 원하는 디자인 스타일의 PPTX/이미지를 input/ 폴더에 넣기

STEP 5-2: Gemini 디자인 프롬프트 생성 (Claude 자동)
  → PPTX 콘텐츠 + 레퍼런스 분석 → 슬라이드별 Gemini 프롬프트 작성

STEP 5-3: Gemini에서 HTML 디자인 제작 (사용자 수동)
  → gemini.google.com에서 프롬프트 복붙 → HTML 생성 → 대화하며 수정

STEP 5-4: Claude에서 워딩/구조 수정 (Claude + 사용자 대화)
  → Gemini HTML을 기획안과 대조하여 텍스트 정밀 수정
  → Action Title, KPI 수치, Win Theme 반영

STEP 5-5: Puppeteer JSON 변환 (자동)
  → node render.js presentation.html → .figma.json 생성

STEP 5-6: Figma 플러그인 임포트 (사용자 수동)
  → Figma에서 플러그인 실행 → .figma.json 드래그 앤 드롭 → 변환
  → 모든 요소가 편집 가능한 네이티브 Figma 프레임으로 변환됨
```

### 결과물

| 산출물 | 설명 |
|--------|------|

## 7. Figma 플러그인 (HTML → Figma 변환)

### 왜 직접 플러그인을 만들었는가?

- html.to.design (h2d) 유료 플러그인의 한계: 슬라이드 수 제한, 비용 발생
- 근본 원인: CSS 레이아웃을 정확하게 Figma로 변환하려면 **실제 브라우저 렌더링**이 필요
- 해결: Puppeteer(Chrome)로 HTML을 렌더링 → `getBoundingClientRect()`로 정확한 좌표 추출 → Figma 플러그인이 절대 좌표로 배치

### 사용법

```bash
# 1. JSON 생성 (Puppeteer가 Chrome으로 HTML 렌더링)
cd figma-plugin
node render.js ../projects/[프로젝트]/output/05_design/html/presentation.html

# 2. Figma 플러그인에서 .figma.json 파일 드롭 → 변환
```

### 플러그인 설치 (최초 1회)

1. Figma 데스크톱 앱 열기
2. Plugins > Development > Import plugin from manifest...
3. `figma-plugin/manifest.json` 선택
4. "Proposal HTML to Figma" 플러그인 등록 완료

## 8. 실습

### 실습 1: 제안서 자동 생성

> RFP를 넣고 5단계 파이프라인으로 PPTX 제안서가 자동 생성되는 과정 확인

1. **RFP 준비**: 샘플 RFP PDF를 `제안요청서/` 폴더에 넣기
2. **에이전트 실행**: Claude Code에서 `/start` → B 선택
3. **5단계 자동 실행**: 분석 → 리서치(스킵 가능) → 기획 → 제작 → QA
4. **결과 확인**:
   - PPTX 파일 열어서 전체 구조 확인
   - Impact-8 Phase 구조, Win Theme 반영 여부
   - Action Title, KPI 산출근거 확인
   - QA 리포트: critical 이슈 0건 확인

### 실습 2: Figma 디자인 변환

> PPTX 이후 고퀄리티 디자인으로 Figma에 임포트하는 과정 시연

1. **STEP 5 진행**: "디자인 진행" 선택
2. **레퍼런스 입력**: 원하는 디자인 스타일 이미지를 input/에 넣기
3. **프롬프트 생성**: Claude가 Gemini용 프롬프트 자동 작성
4. **Gemini에서 HTML 제작**: 프롬프트 복붙 → HTML 생성 → 수정
5. **워딩 수정**: Claude에서 기획안 기준 텍스트 교정
6. **Figma 임포트**: `node render.js` → Figma 플러그인 → 변환
7. **결과 확인**: Figma에서 모든 요소가 편집 가능한 네이티브 프레임

### 실습 3: 브레인스토밍 → 제안서 풀 코스

> 3명의 AI 전문가가 캠페인을 기획하고 제안서까지 자동 생성되는 과정 시연

1. **풀 코스 시작**: `/start` → A 선택
2. **브레인스토밍**: 3명의 AI 전문가가 각자 기획안 제시 → 토론 → 최적안 선택
3. **기획안 완성**: campaign_brief.md 생성
4. **제안서 자동 연계**: BriefAdapter가 브리프를 제안서 형식으로 변환
5. **STEP 3~5 자동 실행**: 기획 → 제작 → 디자인 변환

## 9. 오늘 배운 내용 최종 정리

### 핵심 요약
1. **문제 정의가 먼저다** — 제안서 제작의 반복적 고통(구조 설계, 디자인, 콘텐츠 기획)을 정리하는 것이 출발점
2. **RFP + 문제점을 에이전트에 전달** — 데이터와 요구사항을 함께 주면 멀티 에이전트 파이프라인 자동 생성
3. **Impact-8 구조가 핵심** — HOOK~INVESTMENT 8개 Phase로 제안서의 설득 흐름을 자동 설계
4. **초안 → 반복 개선** — 한 번에 완벽하지 않아도 됨, 부족한 기능을 추가 요청하며 완성
5. **PPTX + Figma 듀얼 출력** — 납품용 PPTX 자동 생성 + Figma 고퀄리티 변환
6. **브레인스토밍 연계** — 3명의 AI 전문가 기획 → 제안서 자동 생성 원스톱
7. **Figma 플러그인 직접 개발** — Puppeteer 렌더링 → 절대 좌표 JSON → Figma 네이티브 변환
8. **제안서 이후가 진짜** — 자동 생성 후 Claude Code로 워딩 수정, 구조 변경, 테마 변경 자유자재

### 실무 활용 루틴
- RFP 수신 → `제안요청서/` 폴더에 넣기 → `/start`
- 5단계 자동 실행 → PPTX 납품용 완성 (15~30분)
- (선택) STEP 5 디자인 변환 → Gemini HTML → Claude 워딩 수정 → Figma 임포트
- 수정 필요 시 → "수정해줘", "테마 변경", "프롬프트 재생성" 등 자연어로 요청
- 브레인스토밍이 필요하면 → 풀 코스(A) 선택 → 기획부터 제안서까지 원스톱
