# 브레인스토밍 에이전트 - 강의 스크립트 구성안

---

## 1. 브레인스토밍 에이전트를 만든 이유

### 캠페인 기획의 페인 포인트

- **혼자 기획하면 시야가 좁아짐**
  - 전략 관점만 강하거나, 크리에이티브만 강하거나, 데이터만 강한 편향된 기획
  - "이게 최선인가?" 확신이 없는 상태로 제안서 작성 시작
  - 다른 관점의 피드백을 받으려면 팀 미팅을 잡아야 하는데 시간/비용 부담

- **팀 브레인스토밍의 비효율**
  - 회의실 잡고, 일정 조율하고, 진행하는 데만 반나절
  - 회의에서 나온 아이디어를 정리하는 데 또 시간 소요
  - 목소리 큰 사람의 의견이 채택되는 경향 (데이터 기반이 아닌 감에 의존)
  - 전략가-크리에이터-데이터 분석가가 한 팀에 있는 경우가 드뭄

- **기획안에서 제안서까지의 갭**
  - 기획안을 만들어도 이걸 제안서 구조로 변환하는 작업이 또 필요
  - Win Theme, Action Title, KPI 산출근거를 기획안에서 제안서로 일일이 옮겨야 함
  - 기획 → 제안서 전환 과정에서 핵심 메시지가 희석되거나 누락

- **벤치마크/데이터 수집의 부담**
  - "업계 평균 CTR이 얼마야?", "경쟁사 SNS 팔로워가 몇이야?" 매번 조사
  - KPI 목표를 설정하려면 근거 데이터가 필요한데 찾기 번거로움
  - 예산 배분을 감으로 하게 됨 (채널별 ROI 시뮬레이션 없이)

### 이 문제를 에이전트로 해결하면?

- **3명의 AI 전문가**가 동시에 독립적으로 캠페인 기획 → 편향 없는 다각도 분석
- 전략가·크리에이터·데이터 분석가가 **서로 토론하며 최적안 도출** (Agent Teams)
- 기획안 완성 → **BriefAdapter로 제안서 자동 연계** (기획→제안서 갭 해소)
- 벤치마크, KPI, 예산 배분을 **데이터 기반으로 자동 산출**
- 기획안을 **웹앱으로 시각화** (클라이언트 프레젠테이션용)
- 전체 과정 **30~50분** 완료 (팀 미팅 대비 1/3 시간)

## 2. 브레인스토밍 에이전트 만든 과정

### Step 1: 문제 사항 정리
- "혼자 기획하면 관점이 편향된다" → 3명의 전문가 관점이 필요
- "토론 없이 기획하면 약점이 드러나지 않는다" → 강제 반론 토론 필요
- "기획안→제안서 전환이 수동이다" → 자동 연계 어댑터 필요
- **이 단계가 가장 중요** — 어떤 전문가가 필요하고, 어떤 토론 구조가 효과적인지 먼저 설계

### Step 2: 3명의 AI 전문가 설계

각 에이전트의 **관점과 프레임워크**를 명확히 정의합니다:

```
전략가 (Campaign Strategist)
  → "어떻게 이겨야 하는가?"
  → PEST 분석 + 3C 분석 + 포지셔닝 맵
  → 시장 환경, 경쟁 구도, 타겟 세분화

크리에이터 (Campaign Creator)
  → "어떻게 보여줘야 하는가?"
  → Big Idea + 메시지 체계 + 채널별 콘텐츠 전략
  → 크리에이티브 컨셉, 톤앤매너, 캠페인 네이밍

데이터 분석가 (Campaign Data Analyst)
  → "어떻게 증명해야 하는가?"
  → 업계 벤치마크 + KPI 설계 + ROI 시뮬레이션
  → 예산 최적화, 시나리오 분석, AARRR 퍼널
```

### Step 3: 토론 구조 설계 (Agent Teams)

단순히 3명이 각자 기획하는 게 아니라 **서로 토론하는 구조**가 핵심입니다:

```
[왜 토론이 중요한가?]
- 각자의 기획안에는 반드시 약점이 있음
- 전략가의 전략이 크리에이티브로 실현 가능한지 크리에이터가 검증
- 크리에이터의 아이디어가 데이터로 뒷받침되는지 분석가가 검증
- 분석가의 KPI가 전략 방향과 일치하는지 전략가가 검증

[토론 3라운드]
Round 1: 교차 리뷰 — 3개 기획안의 공통점 vs 충돌점 식별
Round 2: 강제 반론 — 각 에이전트가 다른 기획안에 반론 제시 (필수!)
Round 3: 통합 최적안 — 합의하여 하나의 최종안 도출
```

### Step 4: 기획안→제안서 연계 설계 (BriefAdapter)

```
[BriefAdapter의 역할]
campaign_brief.md → RFPAnalysis JSON + ResearchResult JSON 자동 변환

- 전략가의 3C 분석 → 경쟁 환경 분석
- 크리에이터의 Win Theme → 수주 전략 메시지
- 분석가의 벤치마크 → KPI 산출근거
- 전략 방향 + 컨셉 → Pain Point / 숨겨진 니즈

→ 제안서 STEP 1~2를 스킵하고 STEP 3(기획)부터 바로 진입
```

### Step 5: 초안 기반으로 기능 추가 요청
- "토론에서 반론이 약하다" → 강제 반론 원칙(contention) 추가
- "기획안 정리가 안 된다" → campaign_brief.md 9개 섹션 구조화
- "클라이언트에게 보여줄 수 없다" → 웹앱 대시보드 자동 생성
- "제안서로 연결이 끊긴다" → BriefAdapter 자동 변환 구현
- "중간에 끊기면 처음부터 다시" → 스마트 재개 기능 추가

## 3. 브레인스토밍 에이전트 사용 방법

### /brainstorm 단축어 사용법

`/brainstorm`은 Claude Code의 **커스텀 슬래시 커맨드(Custom Slash Command)** 기능으로 만든 단축어입니다.

#### 어떻게 만들어졌나?

프로젝트 폴더 안 `.claude/commands/brainstorm.md` 파일이 곧 슬래시 커맨드입니다:

```
프로젝트 루트/
├── .claude/
│   └── commands/
│       └── brainstorm.md   ← 이 파일이 /brainstorm 커맨드
├── output/                  ← 산출물이 저장되는 폴더
└── ...
```

- `.claude/commands/` 폴더 안에 `.md` 파일을 넣으면, **파일 이름이 곧 슬래시 커맨드 이름**이 됩니다
- `brainstorm.md` → `/brainstorm`으로 호출 가능
- 파일 안에는 에이전트에게 전달할 **전체 프롬프트**(7단계 워크플로우, 에이전트 역할, 토론 규칙 등)가 들어 있습니다

#### 실행 방법

Claude Code 채팅창에서 `/brainstorm`을 입력하면 됩니다:

```
┌─────────────────────────────────────────┐
│  Claude Code 채팅창                      │
│                                         │
│  > /brainstorm                          │
│    ↑ 이렇게 입력하면 자동으로 시작!        │
│                                         │
│  (자동완성도 지원 — /br 까지만 쳐도 추천)   │
└─────────────────────────────────────────┘
```

1. Claude Code를 **프로젝트 폴더에서** 열기 (`.claude/commands/`가 있는 폴더)
2. 채팅창에 `/brainstorm` 입력 후 Enter
3. `brainstorm.md` 안의 프롬프트가 자동으로 로드되어 Step 1부터 시작

#### /start에서도 접근 가능

`/start` 커맨드(시작 가이드)를 실행하면 메뉴에서 A(풀 코스)를 선택해 `/brainstorm`과 동일한 워크플로우로 진입할 수도 있습니다.

```
/start → A 선택 (풀 코스) → /brainstorm과 동일
/brainstorm → 직접 실행 (바로 시작)
```

#### 나만의 슬래시 커맨드 만들기

같은 방식으로 다른 커맨드도 만들 수 있습니다:

```
.claude/commands/
├── brainstorm.md     → /brainstorm
├── start.md          → /start
└── my-command.md     → /my-command   ← 새 파일 추가 = 새 커맨드!
```

**핵심**: 복잡한 프롬프트를 매번 타이핑하는 대신, `.md` 파일에 저장해두고 슬래시 커맨드로 한 번에 호출하는 것이 커스텀 슬래시 커맨드의 핵심입니다.

### 7단계 워크플로우

```
Step 1. 인사 & 프로세스 소개
Step 2. 캠페인 정보 수집 → campaign_input.md 저장
Step 3. 3인 에이전트 개별 브레인스토밍 (순차 실행)
  ├─ 3-1. 전략가 → brainstorm_strategy.md
  ├─ 3-2. 크리에이터 → brainstorm_creative.md
  └─ 3-3. 데이터 분석가 → brainstorm_data.md
Step 4. Agent Teams 토론 (3라운드) → debate_log.md
Step 5. 기획안 구조화 → campaign_brief.md (9개 섹션)
Step 6. 웹앱 시각화 → campaign_dashboard.html
Step 7. 제안서 연계 → BriefAdapter → 제안서 STEP 3부터 자동 실행
```

### 각 에이전트의 산출물

| 에이전트 | 산출물 | 핵심 내용 |
|---------|--------|----------|
| **전략가** | `brainstorm_strategy.md` | PEST 분석, 3C 분석, 타겟 페르소나, 포지셔닝, 3개 전략 방향 |
| **크리에이터** | `brainstorm_creative.md` | 3개 크리에이티브 컨셉, Big Idea, 메시지 체계, 채널 콘텐츠 플랜, 톤앤매너 |
| **데이터 분석가** | `brainstorm_data.md` | 업계 벤치마크, 경쟁사 디지털 성과, 전략별 데이터 평가, 예산 배분, KPI 설계 |
| **토론** | `debate_log.md` | 3라운드 토론 기록, 공통점/충돌점, 최종 합의안 |
| **기획안** | `campaign_brief.md` | 9개 섹션 구조화 기획안 (Single Source of Truth) |

### campaign_brief.md 9개 섹션

```
1. 프로젝트 개요 — 브랜드, 목표, 기간, 예산
2. 타겟 & 인사이트 — 페르소나 + 핵심 인사이트 3개
3. 캠페인 전략 — 포지셔닝 + Win Theme 3개
4. 크리에이티브 컨셉 — Big Idea + 슬로건 + 메시지 체계
5. 채널 & 콘텐츠 플랜 — 채널별 역할 + 콘텐츠 유형 + KPI
6. KPI & 성과 목표 — 핵심 KPI + 목표치 + 벤치마크 + 산출근거
7. 예산 배분 — 항목별 비율 + 금액
8. 실행 타임라인 — 주차별 활동 + 채널 + 산출물
9. 리스크 & 대응 — 리스크 + 확률 + 영향도 + 대응 방안
```

### 스마트 재개 기능

## 4. 에이전트 구성 상세

### 전략가 (Campaign Strategist)

**관점**: "어떻게 이겨야 하는가?"
**프레임워크**: PEST + 3C + 포지셔닝 맵

| 분석 항목 | 내용 |
|----------|------|
| PEST 분석 | 정치·경제·사회·기술 4요소 × 트렌드 × 캠페인 시사점 |
| 3C 분석 | Company(자사 강점) / Competitor(경쟁사 3~5개) / Customer(고객 니즈) |
| 타겟 정의 | 핵심 타겟 세그먼트 + 시장 규모 + 미디어 접점 |
| 타겟 페르소나 | 이름, 나이, 직업, 일상, 니즈, 미디어 소비 |
| 포지셔닝 기회 | 시장 공백, 차별화 포인트 |
| 3개 전략 방향 | 각각 근거 + 강점 + 리스크 + 예상 ROI |

### 크리에이터 (Campaign Creator)

**관점**: "어떻게 보여줘야 하는가?"
**원칙**: 한 줄 Big Idea, 채널 네이티브, 참여 유도, 브랜드 보이스 일관성

| 산출물 | 내용 |
|--------|------|
| 3개 크리에이티브 컨셉 | Big Idea(한 문장) + 슬로건 + 비주얼 방향 + 레퍼런스 |
| 메시지 체계 | Key Message → Supporting Message → Proof Point |
| 채널 콘텐츠 플랜 | 채널별 역할 + 콘텐츠 유형 + 포맷 + 빈도 |
| 톤앤매너 가이드 | 보이스 성격 + 말투 + 금지 표현 + 비주얼 무드 |
| 캠페인 네이밍 | 3개 후보 + 의미 + 활용 방안 |

### 데이터 분석가 (Campaign Data Analyst)

**관점**: "어떻게 증명해야 하는가?"
**프레임워크**: AARRR 퍼널 + 미디어 KPI

| 산출물 | 내용 |
|--------|------|
| 업계 벤치마크 | 채널별 CTR, VTR, CPC, CPM 평균치 |
| 경쟁사 디지털 성과 | 검색량, SNS 팔로워, 콘텐츠 반응률, 추정 광고비 |
| 전략별 데이터 평가 | 3개 전략 × 타겟 도달, 예산 효율, 경쟁 강도, ROAS |
| 컨셉×채널 적합도 | 매트릭스: 채널 × 컨셉 적합도 점수 + 근거 |
| 예산 배분 최적화 | 채널 믹스 + 예상 도달, 노출, 전환 + 시나리오 분석 |
| KPI 프레임워크 | 핵심 KPI + 보조 KPI + 목표치 + 측정 방법 + 산출근거 |

## 5. 실습

### 실습 1: 3인 에이전트 브레인스토밍

> 실제 캠페인 브리프를 넣고 3명의 AI 전문가가 독립적으로 기획하는 과정 확인

1. `/start` → A 선택 (풀 코스)
2. 캠페인 정보 입력 (브랜드, 목표, 타겟, 예산)
3. 3명의 에이전트가 순차적으로 기획안 작성
4. 각 에이전트의 산출물(strategy/creative/data) 비교
5. **핵심 포인트**: 같은 브리프인데 관점에 따라 전략이 다르다는 것 확인

### 실습 2: Agent Teams 토론

> 3개 기획안을 놓고 에이전트들이 서로 토론하는 과정 시연

1. Step 4 토론 시작
2. Round 1: 공통점 vs 충돌점 식별
3. Round 2: 각 에이전트가 다른 기획안에 **강제 반론** 제시
4. Round 3: 최종 합의안 도출
5. **핵심 포인트**: 반론을 통해 약점이 드러나고, 더 강한 전략이 만들어짐

### 실습 3: 기획안 → 제안서 자동 연계

> campaign_brief.md가 제안서 파이프라인으로 자동 연결되는 과정 시연

1. Step 7 제안서 연계 시작
2. BriefAdapter가 기획안을 RFPAnalysis + ResearchResult로 변환
3. STEP 1~2 스킵, STEP 3(기획)부터 자동 실행
4. Win Theme, KPI, 전략이 제안서에 일관되게 반영되는지 확인
5. **핵심 포인트**: 기획→제안서 전환이 자동, 핵심 메시지 누락 없음

## 6. 오늘 배운 내용 최종 정리

### 핵심 요약
1. **3명의 AI 전문가 설계가 핵심** — 전략가·크리에이터·데이터 분석가의 관점과 프레임워크를 명확히 정의
2. **토론이 기획의 품질을 높인다** — 강제 반론(contention) 원칙으로 약점을 사전에 발견
3. **9개 섹션 구조화** — campaign_brief.md가 Single Source of Truth
4. **제안서 자동 연계** — BriefAdapter로 기획→제안서 갭 해소
5. **스마트 재개** — 중간에 끊겨도 자동으로 이어서 진행
6. **데이터 기반 의사결정** — KPI, 예산, ROI를 감이 아닌 벤치마크 기반으로 산출

### 실무 활용 루틴
- 캠페인 브리프 수신 → `/brainstorm` 실행
- 3명의 전문가 기획 + 토론 → 30~50분에 기획안 완성
- 기획안 확정 → 제안서 자동 연계 → PPTX + Figma 디자인 완성
- 혼자 기획할 때보다 **3배 넓은 관점**, **데이터 기반 근거**, **약점 사전 보완**
