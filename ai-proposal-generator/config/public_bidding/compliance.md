# 공공입찰 인증·컴플라이언스 가이드

> 한국 공공입찰(나라장터) 사업에서 자주 요구되는 인증·법제·표준.
> Phase 5(MANAGEMENT), Phase 6(WHY US)에서 참조.

## 1. 정보보호·개인정보 관리 인증

### ISMS-P (정보보호 및 개인정보보호 관리체계 인증) ★★★
- **근거**: 정보통신망법 / 개인정보보호법
- **인증기관**: KISA (한국인터넷진흥원)
- **유효기간**: 3년 (매년 사후 심사)
- **공공입찰 활용**: 사실상 필수 — 미보유 시 가산점·평가 불리
- **세부 통계**:
  - 2024 기준 ISMS 약 1,200개사 / ISMS-P 약 200개사
- **포함 내용**: 12개 통제 분야, 80개 통제항목 (관리·물리·기술·운영)

### ISMS (정보보호 관리체계 인증)
- ISMS-P의 "정보보호" 부분만
- **의무 대상**: 매출 1500억 / IT 매출 100억 / 정보보호 100만 / 일평균 이용자 100만 이상

### ISO/IEC 27001 (정보보안 국제표준)
- **인증기관**: BSI, DNV, TÜV 등 국제 인증기관
- **공공입찰 활용**: 가산점, 글로벌 신뢰성

### ISO/IEC 27701 (개인정보 관리 국제표준)
- ISO 27001의 확장 — 개인정보 관리시스템(PIMS)
- **공공입찰 활용**: 개인정보 처리 사업 가산점

### PIPL (개인정보보호 인증, 행안부)
- 한국형 개인정보 보호 인증 (구 PIMS)
- **현재**: ISMS-P로 통합 (2018)

## 2. 클라우드 인증

### CSAP (Cloud Security Assurance Program) ★★★
- **근거**: 행정·공공기관 정보자원 통합기준
- **인증기관**: KISA
- **3등급 체계 (2024.1 신설)**:
  - **下 등급**: 비공개 정보 미처리 (대부분 SaaS)
  - **中 등급**: 비공개 + 일반 행정정보
  - **上 등급**: 비공개 + 민감 행정정보 (국방·치안 제외)
- **유형**: IaaS / SaaS / PaaS / 보안서비스
- **공공입찰**: 공공 클라우드 사용 시 필수
- **국산 CSP**: NHN Cloud, KT Cloud, Naver Cloud, Samsung SDS, NIPA G-Cloud

### CC인증 (Common Criteria, EAL 등급) ★★
- **근거**: 공통평가기준 (ISO/IEC 15408)
- **인증기관**: NIS (국가정보원), TTA, KSEL, IT연구원
- **EAL 1~7 등급** (한국 공공: 보통 **EAL2 또는 EAL2+**)
- **공공 의무 대상**: 보안 SW (방화벽·IPS·DRM·보안USB·VPN·DLP 등)
- **참고 사이트**: itscc.kr (한국정보보호제품평가센터)

### CCC (CC평가·인증, 국내) — 보안기능확인서
- **근거**: 정보보안 제품 평가 가이드
- **인증기관**: 국정원
- **공공 도입 시 필수** (CC인증 미보유 보안 제품)

### KCMVP (한국 암호 모듈 검증)
- **인증기관**: 국정원
- **공공 도입 의무**: 행정·공공기관 암호 모듈
- **검증 알고리즘**: SEED, ARIA, LEA 등 한국 표준

## 3. SW·시스템 품질 인증

### GS인증 (Good Software, 굿소프트웨어) ★★
- **인증기관**: TTA, KTL
- **공공입찰**: 가산점 (정보화사업 1~2점)
- **국산 SW 활용 정책**: GS 1등급 SW 우선 도입
- **혜택**: 조달청 종합쇼핑몰 등록, 입찰 가산점, 인지도

### GS인증 등급
- 1등급 / 2등급 (GS 마크)
- 평가 기준: 기능·신뢰·이식·사용·효율·유지보수·보안성

### CMMI (Capability Maturity Model Integration)
- **Lvl. 1~5** (보통 Lvl.3+ 요구)
- **공공 대형 SI 사업**: Lvl.3 이상 필수 (50억 이상 대형)
- **인증기관**: SEI 공인 평가사

### ISO 9001 (품질경영 시스템)
- **공공입찰**: 기본 가산점

### CMMI-SVC (서비스 영역) / DEV (개발 영역)
- IT 서비스·SI 사업 구분

### TTA 인증 (정보통신단체표준)
- **TTAK** 표준 적합 — VMS, AI, IoT 등 분야별
- **공공입찰**: 가산점·기술 적합성 입증

## 9. 평가위원 시각 — "이 회사는 검증되었나?"

**확인 흐름**:
1. **사업자 자격** (SW사업자·정통공사업) — 입찰 자격
2. **인증 보유** (ISMS-P·CSAP·CMMI·GS) — 기본 신뢰
3. **유사 실적** (5건+ 정량) — 수행 능력
4. **핵심 인력** (기술사·PMP) — 실무 능력
5. **재무 안정성** (신용평가) — 계약 이행 능력

→ Phase 6에서 1페이지 인증·자격 매트릭스로 한눈에 보이도록.
