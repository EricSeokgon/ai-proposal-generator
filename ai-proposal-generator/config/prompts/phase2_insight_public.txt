## 목적
"발주처의 정책·과제·Pain Point를 정확히 이해하고 있다"를 증명하여 평가위원의 신뢰 획득.
Phase 2는 평가표의 **"사업 이해도"(보통 5~10점)** 항목과 직결됩니다.

## 핵심 원칙: 정책 → 과제 → 기회 연결
- 단순 시장 트렌드 ❌ → **국정과제·발주처 정책·법제 변화**와 연결 ✅
- 일반 IT 트렌드 ❌ → **본 사업 도메인(빅데이터/AI/인프라/CCTV/데이터)** 특화 분석 ✅
- 추상적 문제 ❌ → **As-Is 현황 진단 (정량 지표)** + RFP 요구사항 추적 ✅

## 작성 가이드

### 2.1 정책·법제 환경 분석 (★공공 특화)
이 사업이 추진되는 **정책적 배경**을 명확히:
- **국정과제 연계**: 디지털플랫폼정부 / 데이터 댐 / K-Cloud / 스마트시티 / AI 일상화 등
- **발주처 중장기 계획**: 발주기관의 「OO 5개년 계획」 또는 「OO 마스터플랜」
- **관련 법제 변화**:
  - 빅데이터: 데이터3법 / 데이터산업법 / 마이데이터
  - AI: AI기본법 / 신뢰성 기준 / EU AI Act
  - 인프라: 클라우드법 / CSAP / 망분리 의무
  - CCTV: 개인정보보호법 25조 / 스마트도시법 / 5대 안전망
  - 거버넌스: 데이터3법 / 가명정보 가이드라인
- 예시:
  - "디지털플랫폼정부 G-클라우드 전환 정책 (2025년 모든 공공기관)"
  - "AI 기본법 시행으로 신뢰성 검증 의무화 (2025.7~)"
  - "5대 안전망 통합플랫폼 100% 연계 의무 (스마트도시법)"

### 2.2 시장·기술 트렌드 (★도메인별 특화)
도메인 카드를 참조하여 **본 사업 도메인의 기술 트렌드**:
- **빅데이터**: 레이크하우스(Iceberg/Delta) / 실시간 스트리밍 / 데이터메시
- **AI**: 한국어 sLLM / RAG / On-prem LLM / 멀티모달 / Agent
- **인프라**: K-Cloud 전환 / 컨테이너화 / FinOps / 제로트러스트
- **CCTV**: 지능형 영상분석(이상행동) / 엣지 AI / 5G 영상 / 디지털 트윈
- **거버넌스**: 가명결합 / 차분 프라이버시 / 데이터 메시 / 자동 분류
- 예시:
  - "공공 LLM 시장 2024년 5,000억 → 2027년 1.8조 (연 53% 성장, NIA)"
  - "지능형 CCTV 도입률 2024년 35% → 2027년 80% 목표 (행안부)"

### 2.3 발주처 As-Is 진단 (★Pain Point)
RFP의 "배경/목적/필요성" 섹션에서 발주처가 토로하는 문제를 추출하여 정량화:
- **데이터 분절**: "XX 시스템 12개 산재, 통합 분석 불가"
- **노후 인프라**: "주요 시스템 평균 가동연수 8년, 장애율 증가"
- **인력 부재**: "데이터 분석 전담 인력 0명, 외주 의존"
- **보안 취약**: "최근 3년간 침해사고 N건, ISMS-P 미보유"
- **법제 미준수**: "가명처리 미시행으로 데이터 활용 불가"
- 각 문제마다 **출처(RFP 페이지) + 영향(정량) + 시급성** 명시

### 2.4 RFP 요구사항 완벽 대응 매트릭스 (★평가위원 핵심 확인 항목)
RFP의 모든 요구사항이 제안서에 반영되었음을 시각적으로 증명:
```
| ID | 요구사항 | RFP 페이지 | 우선순위 | 대응 방안 | 제안서 위치 |
|----|---------|-----------|---------|---------|-----------|
| FR-01 | 실시간 영상 분석 | p.15 | 필수 | GPU 기반 AI 엔진 | Phase4 슬라이드 23 |
| NFR-02 | 가용성 99.9% | p.18 | 필수 | Active-Standby 이중화 | Phase4 슬라이드 31 |
```
→ 평가위원이 1분 만에 "누락 없음"을 확인할 수 있게.

### 2.5 숨겨진 니즈 발굴 (인사이트)
RFP에 명시되지 않은, 발주처 입장의 숨은 니즈:
- "단순 도입이 아닌 운영 인력 양성까지 (이전 사업 인수인계 부실 학습)"
- "장기적 확장성 (3년 후 N배 데이터 처리)"
- "타 부처 연계 (데이터 표준화로 후속 사업 토대)"
- "예산 절감 (TCO 분석 + 국산 솔루션 우선)"

### 2.6 To-Be 비전 — 변화 후 그림
- As-Is → To-Be 비교표 (정량 지표 변화)
- 예시:
  ```
  데이터 처리량   1,000건/sec  →  10만건/sec  (100배)
  분석 응답시간   30분         →  3초          (600배)
  데이터 표준 준수 30%          →  95%
  보안 등급       미보유        →  ISMS-P + CSAP 中
  ```
