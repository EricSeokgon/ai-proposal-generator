### 4.2 추진 일정 (WBS) ★★★
- **단계 구조**: 분석 / 설계 / 개발 / 시험 / 이행 / 안정화
- **마일스톤**: 단계별 종료 산출물 + 중간 보고
- **WBS 200+ 항목**: 작업분해 → 기간 → 담당 → 산출물 → 선후행
- **간트 차트** 시각화

### 4.3 시스템 아키텍처 (★도메인별 특화)
도메인 카드의 표준 아키텍처를 참조하여:
- **빅데이터**: 수집(Kafka) → 저장(Lakehouse) → 처리(Spark) → 분석(Trino) → 서비스(API)
- **AI**: 데이터(Label) → 학습(GPU/MLflow) → 평가(Bench) → 배포(Triton/vLLM) → 모니터링
- **인프라**: 네트워크 → 가상화/K8s → 보안 → 백업/DR → 모니터링
- **CCTV**: 카메라 → 전송망 → VMS → AI 분석 → 5대 안전망 연계
- **거버넌스**: 메타데이터 → 카탈로그 → 품질 → 보안 → 활용

각 아키텍처는 **3단 도식** 필수:
- Logical (기능 구성)
- Physical (서버·네트워크 구성)
- Network (네트워크 토폴로지)

### 4.4 단계별 상세 추진 계획

#### 4.4.1 분석 단계 (5~10%)
- 요구사항 분석 (RFP 100% 추적 + 인터뷰 + 워크숍)
- 현황 분석 (As-Is 시스템·데이터·프로세스)
- 표준·법제 적합성 분석 (KS·TTA·ISMS-P·법규)
- 산출물: **요구사항 정의서 / RTM / 현황분석서 / 표준적합성검토서**

#### 4.4.2 설계 단계 (15~20%)
- **시스템 설계**: 아키텍처·DB·UI·인터페이스
- **상세 설계**: 화면·테이블·API
- **보안 설계**: 망구성·접근통제·암호화·감사로그 (★ 필수)
- 산출물: **시스템설계서 / DB설계서 / 화면설계서 / 인터페이스정의서 / 보안설계서**

#### 4.4.3 구현·개발 단계 (40~50%)
- 도메인별 핵심 작업:
  - **빅데이터**: 데이터 표준화 / 파이프라인 / 카탈로그 / 품질
  - **AI**: 데이터 라벨링 / 모델 학습 / MLOps / 신뢰성 검증
  - **인프라**: 서버·네트워크 구축 / 마이그레이션 / 이중화
  - **CCTV**: 카메라 설치 / VMS / AI 분석 / 5대 안전망 연계
  - **거버넌스**: 메타데이터 등록 / DQM / 비식별화 / 카탈로그
- 산출물: **단위시험결과서 / 통합시험결과서 / 형상관리대장**

#### 4.4.4 시험 단계 (10~15%)
- **단위시험**: 모듈별 기능 검증
- **통합시험**: 모듈 간 인터페이스
- **시스템시험**: 성능·부하·보안·장애 복구
- **인수시험(UAT)**: 발주처 검증
- 산출물: **시험계획서 / 시험시나리오 / 시험결과서 / 시험증적**

#### 4.4.5 이행·안정화 단계 (10~15%)
- 운영 이관 (Run Book)
- 사용자 교육 (관리자 / 운영자 / 일반 사용자)
- 안정화 운영 (보통 3~6개월)
- 산출물: **운영매뉴얼 / 사용자매뉴얼 / 교육교재 / 인수인계서**
