# 도메인 카드 — 빅데이터 / 데이터플랫폼

## 정책·법제 연계 (한국 공공)
- **데이터 산업법** (2022.4.20 시행) — 데이터 자산 가치, 데이터 거래
- **데이터 3법** — 개인정보보호법 / 정보통신망법 / 신용정보법 (가명정보 활용)
- **공공데이터법** — 공공데이터 개방·활용 의무
- **마이데이터(MyData)** — 금융·의료·공공 영역 본인정보 이동권
- **디지털플랫폼정부** — 정부24, 안전신문고 등 통합 데이터 인프라
- **K-Data 산업진흥원** — 데이터바우처, 데이터 가치평가

## 표준 아키텍처 (전형 참조도)
```
[원천] → [수집] → [저장] → [처리/가공] → [분석/활용] → [서비스]
 OLTP    Kafka    Data Lake  Spark/Flink   Trino/      Tableau/
 IoT     NiFi     (HDFS/S3)  Airflow       Presto       Superset
 Open    Logstash Data       dbt           ML Studio    REST API
 API              Warehouse  Iceberg       Notebook     OpenData
                  (Postgres                              포털
                   /Snowflake)
```

핵심 컴포넌트:
- **수집**: Kafka, NiFi, Logstash, Beats, Fluentd
- **저장**: HDFS, MinIO/S3, Iceberg/Delta Lake (레이크하우스), PostgreSQL, ClickHouse
- **처리**: Spark, Flink, Airflow, dbt, Trino/Presto
- **분석**: Jupyter, Zeppelin, Superset, Metabase, Tableau, Power BI
- **거버넌스**: Apache Atlas, DataHub, Amundsen, Collibra
- **품질**: Great Expectations, Soda, Apache Griffin

## 기술 요구사항 체크리스트
- [ ] 데이터 표준화 (DDM 데이터 모델, 메타데이터 사전)
- [ ] 데이터 품질 관리 (정확성·완전성·일관성·적시성·유효성·접근성)
- [ ] 데이터 카탈로그 (계보 추적, 검색)
- [ ] 데이터 보안 (접근통제, 암호화, 비식별화)
- [ ] 확장성 (수평/수직 스케일, 클라우드 네이티브)
- [ ] 표준 API (RESTful, OpenAPI 3.0, GraphQL)
- [ ] 모니터링 (Prometheus + Grafana, ELK)

## 필수 인증·표준
- **ISMS-P** (정보보호 및 개인정보보호 관리체계 인증)
- **CSAP** (클라우드 보안 인증) — 공공기관 클라우드 사용 시 필수
- **GS인증** (국산 SW 품질) — 정보화사업 가산점
- **TTA 인증** (정보통신단체표준)
- **개인정보 영향평가(PIA)** — 5만 명 이상 개인정보 처리 시 의무

## 유사 사업 레퍼런스 (제안서 인용 가능 사례)

## Win Theme 후보 (이 도메인용)
1. **"표준 기반 통합"** — 부처/기관 간 데이터 표준화로 사일로 해소
2. **"안전한 데이터 활용"** — 가명처리·암호화 기반 개인정보 보호 + 활용 확대
3. **"검증된 운영"** — 유사 공공 사업 실적 기반 안정적 운영 보증
