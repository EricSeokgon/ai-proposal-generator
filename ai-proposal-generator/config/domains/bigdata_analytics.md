## 표준 분석 방법론

### 1. 분석 라이프사이클
```
[기획]            [데이터]              [모델링]         [활용]
 과제 정의 →      수집 → 정제 → 변환 → EDA → 모델링 → 평가 → 적용 → 모니터링
 KPI 설정         (전체의 60%+ 시간)   (15~20%)         (정책 적용 + 효과 측정)
```

### 2. 분석 유형
- **기술적 분석 (Descriptive)**: "무엇이 일어났는가" — 통계·집계·시각화
- **진단 분석 (Diagnostic)**: "왜 일어났는가" — 상관·인과·드릴다운
- **예측 분석 (Predictive)**: "무엇이 일어날 것인가" — ML 회귀·분류·시계열
- **처방 분석 (Prescriptive)**: "무엇을 해야 하는가" — 최적화·시뮬레이션

### 3. 공공 분석 단골 프레임워크
- **PPDAC**: Problem → Plan → Data → Analysis → Conclusion
- **CRISP-DM**: Cross-Industry Standard Process for Data Mining
- **OSEMN**: Obtain → Scrub → Explore → Model → iNterpret

## 핵심 분석 도구·기법

### 통계 분석
- **R / Python**: pandas, scipy, statsmodels
- **상용**: SAS, SPSS, Stata, Minitab

### 머신러닝 / 딥러닝
- **scikit-learn, XGBoost, LightGBM**: 정형 데이터
- **PyTorch, TensorFlow**: 딥러닝
- **Prophet, ARIMA, LSTM**: 시계열 예측

### 공간 분석 (★공공 빅데이터 단골)
- **GIS**: QGIS, ArcGIS, PostGIS
- **공간 통계**: Moran's I, Getis-Ord, Hot Spot 분석
- 활용: 유동인구·상권·범죄·교통·재난

### 텍스트 분석 (NLP)
- **국문 NLP**: KoNLPy, KLUE, KoELECTRA, KoBERT
- 활용: 민원·소셜 분석·정책 평가

### BI·시각화
- **상용**: Tableau, Power BI, Qlik, Looker
- **오픈소스**: Superset, Metabase, Grafana, Plotly Dash
- **국산**: 와탭, 크로센트(Brightics), DataDeli, MyDataSpot

## 분석 보고서 작성 표준 (Phase 4 산출물 가이드)

### 표준 구조
1. Executive Summary (1~2p) — 핵심 인사이트 + 정책 제안
2. 분석 배경 / 목적 / 범위
3. 데이터 명세 / 한계
4. 분석 방법론
5. 분석 결과 (시각화 + 통계)
6. 인사이트 도출 (★)
7. 정책 제안 (★ 평가위원 핵심 확인)
8. 부록 (코드 / 상세 표)

### 시각화 원칙
- **공공 분석은 시각화가 80%** — 평가위원이 그림으로 이해
- 한 슬라이드 = 한 인사이트 + 핵심 차트
- 색상은 의미 있게 (감정 색상 회피, 색맹 고려)
- 출처·기준일자 명시 필수
