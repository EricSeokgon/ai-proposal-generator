## 고도화 사업 4대 유형

### 1. 기능 고도화 (Functional Enhancement)
- 기존 기능 개선 + 신규 기능 추가
- 예: 배치 → 실시간 / 룰 기반 → AI / 단순 BI → 셀프 서비스 BI
- **표준 산출물**: 기능 영향도 분석서, 기능 매핑표

### 2. 기술 현대화 (Technology Modernization)
- 기술 스택 전환 (Hadoop → Spark, Oracle → PostgreSQL, VM → K8s)
- 모놀리스 → 마이크로서비스 / 데이터 → 레이크하우스
- **표준 산출물**: 기술 매핑표, 호환성 검증 보고서

### 3. 인프라 전환 (Infrastructure Migration)
- 온프레미스 → 클라우드 / IDC 이전 / K-Cloud 전환
- **표준 산출물**: 마이그레이션 계획, 데이터 이관 검증서

### 4. 통합 플랫폼화 (Platform Integration)
- 분산 시스템 → 통합 플랫폼
- 부처/기관 간 데이터 표준화 + 통합
- **표준 산출물**: 통합 아키텍처, 표준 사전, 연계 매트릭스

## 고도화 표준 방법론 — 6단계 안전 전환

```
[현황진단]      [TO-BE 설계]    [전환계획]      [병행운영]      [전환]         [안정화]
 As-Is 분석     아키텍처 재설계  마이그레이션    Old/New 동시   Cut-Over       3~6개월
 Gap 분석       데이터 매핑      위험·롤백       데이터 동기화   D-Day          모니터링
 의존성 분석    영향도 평가      검증 시나리오   기능 비교 검증  롤백 대비     점진 폐기
```

### 핵심 원칙
1. **As-Is 100% 이해 후 TO-BE 설계** — 모르는 의존성 = 사고
2. **점진적 전환** (Big Bang 지양) — 모듈/도메인 단위 단계 이행
3. **무중단 보장** — 병행운영 또는 Blue-Green / Canary
4. **롤백 계획 필수** — 문제 시 즉시 복원
5. **데이터 정합성 검증** — 신구 시스템 결과 일치 확인

## 핵심 전환 패턴

### 데이터 이관 패턴
| 패턴 | 설명 | 적용 |
|------|------|------|
| **빅뱅 (Big Bang)** | 한 번에 전체 이관 | 소규모, 재구축형 |
| **점진적 (Incremental)** | 단계별 부분 이관 | 대규모, 무중단 |
| **병행 운영 (Parallel Run)** | 신구 동시 가동 | 중요 시스템, 검증 |
| **CDC 기반 (Change Data Capture)** | 실시간 동기화 | 무중단 필수 |
| **Strangler Fig** | 점진 교체 (마이크로서비스화) | 모놀리스 → MSA |

### 클라우드 전환 패턴 (6R)
| 패턴 | 설명 | 사례 |
|------|------|------|
| **Rehost** ("Lift & Shift") | VM 그대로 클라우드 이전 | 빠른 전환 |
| **Replatform** | 일부 최적화 (DB → RDS 등) | 균형 |
| **Refactor** | 클라우드 네이티브 재설계 | 장기 ROI |
| **Repurchase** | SaaS로 교체 | ERP·CRM |
| **Retain** | 보류 (특수 시스템) | 레거시 잔존 |
| **Retire** | 폐기 | 미사용 시스템 |

### 데이터 검증 (★고도화 핵심 산출물)
- **건수 검증**: 신구 행 수 일치
- **합계 검증**: 주요 지표 합계 일치
- **샘플 검증**: 무작위 N건 1:1 비교
- **회귀 검증**: 기존 보고서·쿼리 결과 일치
- **성능 검증**: TO-BE가 AS-IS 대비 N배 개선

## 자주 보는 감점 요인 (고도화 특화)
❌ As-Is 분석 부실 → 전환 중 의존성 발견으로 지연
❌ 롤백 계획 없음 → "전환 실패 시 어떻게 할 것인가" 미답
❌ 정합성 검증 막연 → 평가위원 신뢰 상실
❌ 병행운영 기간 부족 → "1주 운영 후 전환" 같은 비현실
❌ 데이터 이관 자동화 없음 → 수작업 = 사고 위험
❌ 사용자 영향 분석 없음 → 가동 후 거부감·VOC
