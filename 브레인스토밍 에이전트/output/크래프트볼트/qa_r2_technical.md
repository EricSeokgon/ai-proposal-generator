## Round 1 수정 확인

### 1. 슬라이드 ID 순차 정렬 및 총 수 표기 수정

**결과: 미완료 — 불일치 잔존**

- HTML `id` 속성: `slide-01` ~ `slide-40` (40개) 순서 완전 정상
- 슬라이드 번호 표기: **전 슬라이드 `XX / 40`** 으로 통일됨 — 기계적 표기 자체는 이상 없음
- **문제:** 스크립트(proposal_script.md) 메타데이터에는 "총 슬라이드: 38장"으로 명기되어 있고, 인덱스(부록 A)에는 41개 항목(1-01 ~ 6-41)이 열거됨. 그런데 HTML 실제 구현은 40개임.
  - 스크립트 번호 `4-30`이 결번(스킵)으로 스크립트에는 없음
  - 스크립트 41개 슬라이드 기획 → HTML 40개 구현 (1개 누락 또는 의도적 병합)
  - 메타데이터의 "38장" 수치는 스크립트 자체 오류(이미 R1 이전부터 존재)
- **권고:** 스크립트 메타데이터 "총 슬라이드 38장" → "40장" 으로 수정 필요. HTML 표기(`01 / 40`)는 올바름.

### 2. Chart.js annotation 플러그인 추가 여부

**결과: 미완료 — 플러그인 CDN 미삽입**

- `<head>` 스크립트 태그에 `chartjs-plugin-annotation` CDN 링크가 **존재하지 않음**
  - 현재 로드 중인 스크립트: `chart.js@4.4.1`, `html2canvas`, `jszip`
- chart-06 옵션에 `annotation: {}` 빈 객체가 선언되어 있음 (라인 2439)
- 플러그인 없이 `annotation: {}` 옵션만 존재하면 Chart.js 4.x에서는 **경고 없이 무시**되므로 차트 렌더링은 정상이나, annotation 기능(배경 밴딩 등)은 비활성 상태
- **권고:** 실제 annotation이 필요하지 않다면 `annotation: {}` 옵션 제거로 코드 정리. 배경 밴딩이 필요하면 CDN 추가 필수.

```
<!-- 추가 필요 -->
<script src="https://cdn.jsdelivr.net/npm/chartjs-plugin-annotation@3/dist/chartjs-plugin-annotation.min.js"></script>
```

### Chart-33: 시나리오 그룹 바 차트 (slide-32)

**데이터 정합성: PASS (수치), 단 chart ID 불일치 주의**

- 스크립트 지정 시나리오 데이터:
  - 보수적: `[4000, 6500, 8000]`
  - 기본: `[6000, 10000, 11500]`
  - 낙관적: `[8000, 12500, 15000]`
- HTML 실제 값: 동일 — 완전 일치

**단, canvas 요소 ID 불일치:**
- slide-32(매출 시나리오 슬라이드)에 배치된 canvas ID: `chart-33`
- 슬라이드 번호(32)와 canvas ID(33)가 1 어긋남
- JavaScript 참조 변수명 `ctx33`도 일관되게 "33"을 사용하므로 기능상 문제는 없으나 네이밍 혼란 존재

**매출 합계 검증:**
- 보수적: 4000+6500+8000 = 1.85억 ✓
- 기본: 6000+10000+11500 = 2.75억 ✓
- 낙관적: 8000+12500+15000 = 3.55억 ✓

**슬라이드 제목 vs 스크립트 차트 스펙 차이:**
- 스크립트는 "광고 매출"과 "자연 매출"을 `3500/5000/5500 + 2500/5000/6000` 으로 스택드 바 구성 가능하도록 언급했으나, HTML은 단순 **시나리오 3종 그룹 바**로 구현
- 스크립트의 해당 차트 스펙(`chart_spec: { type: "grouped_bar" }`)과 HTML 구현은 일치하므로 치명적 오류는 아님. 다만 "광고/자연" 분리 시각화는 구현되지 않음.
