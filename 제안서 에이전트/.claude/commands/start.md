# 제안서 에이전트 시작 가이드

아래 가이드에 따라 사용자에게 제안서 제작 과정을 안내합니다. 반드시 한국어로 응답하세요.

---

## 시작 인사

다음과 같이 인사한 후 **진행 방식**을 먼저 질문합니다:

```
안녕하세요! 👋 제안서 자동 생성 에이전트입니다.

RFP(제안요청서)나 기획안을 분석하여
프로페셔널한 PPTX 제안서를 자동으로 제작해 드립니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🚀 진행 방식을 선택해 주세요:

  A. 브레인스토밍 → 제안서 (풀 코스)
     → 브레인스토밍 에이전트로 캠페인 기획 후 제안서 제작
     → 3명의 AI 전문가(전략가·크리에이터·데이터분석가)가 기획
     → 기획안이 완성되면 자동으로 제안서 제작으로 연계
     → 소요: 약 30~50분

  B. 제안서 바로 제작 (단독 실행)
     → RFP/기획안 파일을 바로 분석하여 제안서 제작
     → 5단계 자동 실행: 분석 → 리서치 → 기획 → 제작 → 변환
     → 소요: 약 15~30분

어떤 방식으로 진행할까요? (A 또는 B)
```

## 방식 A: 브레인스토밍 → 제안서 (풀 코스)

### A-1. 브레인스토밍 시작

```
✅ 풀 코스를 선택하셨습니다!

먼저 브레인스토밍 에이전트로 캠페인 기획을 진행합니다.
완성된 기획안(campaign_brief.md)을 기반으로 제안서를 제작합니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 풀 코스 프로세스
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  🧠 Phase 1: 브레인스토밍 (7단계)
  ├─ Step 1~2: 정보 수집
  ├─ Step 3: 3인 에이전트 브레인스토밍
  ├─ Step 4: Agent Teams 토론
  ├─ Step 5~6: 기획안 + 웹앱 시각화
  └─ Step 7: 제안서 연계 ← 자동 전환

  📄 Phase 2: 제안서 제작 (3단계 — 분석/리서치 스킵)
  ├─ STEP 3: 기획 (브리프 기반)
  ├─ STEP 4: 제작 + 검수
  └─ STEP 5: 변환 (선택)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🧠 브레인스토밍을 시작합니다!
```

이후 `/brainstorm` 커맨드의 Step 1부터 진행합니다.

### A-2. 브레인스토밍 → 제안서 전환 (Step 7)

브레인스토밍 Step 6(웹앱 시각화)이 완료되면 자동으로 제안서 제작을 안내합니다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔄 Phase 2: 제안서 제작으로 전환
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

브레인스토밍이 완료되었습니다! 🎉

📋 확정된 기획안:
  • 전략: [최적안 전략명]
  • 컨셉: [크리에이티브 컨셉명]
  • Win Theme 3개 확정
  • KPI 프레임워크 수립 완료

이제 이 기획안을 PPTX 제안서로 제작합니다.

⚡ 브레인스토밍에서 이미 완료된 작업:
  ✅ 분석 (전략가가 시장/타겟/경쟁 분석 완료)
  ✅ 리서치 (데이터 분석가가 벤치마크 수집 완료)
  → STEP 1~2를 건너뛰고 STEP 3(기획)부터 시작합니다

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

이때 `BriefAdapter`(src/agents/brief_adapter.py)를 사용하여:
1. `campaign_brief.md` → RFPAnalysis 형식으로 변환
2. `brainstorm_strategy.md` + `brainstorm_data.md` → ResearchResult 추출
3. Stage 3(기획)부터 파이프라인 실행

### A-3. 제안서 유형 + 테마 선택

브리프 변환 후 제안서 유형과 디자인 테마를 물어봅니다:

```
📋 제안서 설정:

  제안서 유형: 마케팅 제안서 (브리프 기반 자동 판별)
  → 변경하시려면 말씀해주세요

🎨 디자인 테마를 선택해 주세요:

  1. warm_minimal  — 따뜻한 베이지 톤 (기본)
  2. classic_blue  — 클래식 네이비 블루
  3. forest        — 포레스트 그린 (자연/ESG)
  4. corporate     — 코퍼레이트 네이비 (금융/공공)
  5. mono_black    — 모노 블랙 (미니멀/IT)
  6. soft_purple   — 소프트 퍼플 (크리에이티브)

  💡 디자인 레퍼런스 이미지가 있으면 input/ 폴더에 넣어주세요
```

## STEP별 놓치면 안 되는 포인트

### STEP 1: 분석
- 평가기준 배점이 높은 항목 = 제안서에서 가장 비중 있게 다뤄야 할 부분
- RFP '배경/목적' 섹션에 발주처의 진짜 고민이 숨어있음
- **마케팅**: 타겟 오디언스, 채널 전략 방향, 캠페인 목표 중점
- **입찰**: 평가기준 배점, 필수 요구사항, 산출물 목록 중점

### STEP 2: 리서치
- 스킵 가능 (리서치 없이도 제안서 생성 가능)
- KPI 산출근거에 활용할 벤치마크 데이터가 핵심
- **마케팅**: SNS 트렌드, 경쟁 브랜드 캠페인, 채널별 KPI 벤치마크
- **입찰**: 시장 규모, 유사 입찰 낙찰 사례, 기술 트렌드

### STEP 3: 기획
- Action Title: "타겟 분석" (X) → "MZ세대 하루 SNS 55분, 릴스가 핵심" (O)
- 연속 3장 같은 레이아웃 금지
- Win Theme이 Phase 전체에 일관되게 반영되어야 함

### STEP 4: 제작+검수
- QA 검수에서 critical 이슈가 나오면 자동 재생성 (최대 2회)
- Pretendard 폰트만, C["primary"] 등 테마 상수만 사용
- 결과물: PPTX (납품용) + QA 리포트

### STEP 5: 디자인 변환 (고품질 HTML → Figma)
- **선택 사항** — PPTX만 필요하면 스킵 가능
- DesignAgent가 슬라이드별 프롬프트 패키지를 자동 생성
- 사용자가 Gemini 웹(2.5 Pro)에서 직접 HTML 제작
- Puppeteer로 HTML 렌더링 → `.figma.json` 생성 (`node render.js`)
- Figma 플러그인(Proposal HTML to Figma)으로 임포트
